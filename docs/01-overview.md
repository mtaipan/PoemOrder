# Обзор проекта

## Назначение

PoemOrder — персональный сайт автора, который пишет стихи на заказ (поздравления, признания, свадьбы, юбилеи). Сайт решает три задачи:

1. **Витрина** — показать опубликованные работы, цены и условия, отзывы клиентов.
2. **Приём заявок** — клиент заполняет форму (имя, телефон, соцсеть, описание идеи), автор получает уведомление в Telegram и связывается сам.
3. **Самообслуживание автора** — весь контент (стихи, тексты страниц, контакты, цены) редактируется через админку без программиста.

Сознательно **нет**: онлайн-оплаты, личных кабинетов клиентов, корзины. Сделка завершается в переписке — сайт только генерирует лиды.

## Роли

| Роль | Возможности |
|---|---|
| Посетитель | Просмотр всех публичных страниц, отправка заявки, отправка отзыва (уходит на модерацию) |
| Администратор | Всё содержимое сайта: стихи, заказы, модерация отзывов, контакты, настройки текстов |

Администраторов может быть несколько (например, автор и помощник) — все с одинаковой ролью `ADMIN`, задаются переменной `APP_ADMINS`.

## Функциональность

### Публичная часть

- **Главная `/`** — hero-блок (заголовок и подзаголовок из настроек), три последние опубликованные работы, три последних одобренных отзыва.
- **Портфолио `/portfolio`** — все опубликованные стихи; у каждого название, отрывок (или авто-обрезка текста до 300 символов) и ссылка на полный текст `/portfolio/{id}`.
- **Цены `/pricing`** — заголовок и три текстовых блока («Оплата», «Доставка», «Возврат») из настроек; многострочный текст рендерится поабзацно.
- **Отзывы `/reviews`** — список одобренных отзывов + форма отправки. Можно указать Telegram-username и галочкой разрешить его публикацию.
- **Контакты `/contacts`** — карточки контактов из админки: подпись, значение и опциональная кликабельная ссылка.
- **Заявка `/order`** — форма заказа с валидацией; после отправки — flash-сообщение об успехе.

### Админ-панель `/admin`

- **Стихи** — создание/редактирование/удаление; статусы Черновик/Опубликовано; на публику попадают только опубликованные.
- **Заказы** — список с пагинацией (20/стр., новые сверху), карточка заказа, смена статуса: `NEW → IN_PROGRESS → DONE / CANCELED`.
- **Отзывы** — очередь модерации (целиком) и архив опубликованных (пагинация); действия: принять, скрыть контакт, удалить.
- **Контакты** — CRUD карточек: подпись, значение, ссылка (валидируется схема), порядок сортировки, вкл/выкл.
- **Настройки** — hero-тексты главной, заголовок/подзаголовок портфолио, тексты ценовых блоков, легаси-поля контактов.

### Служебная

- Telegram-уведомления о новых заявках и отзывах (опционально, см. [09-telegram.md](09-telegram.md)).
- Антиспам: rate limiting + honeypot (см. [08-antispam.md](08-antispam.md)).
- Health-check `GET /actuator/health` для мониторинга/оркестрации.

## Чего в проекте осознанно нет

- **SPA/JS-фреймворков** — сервер-сайд рендеринг покрывает все сценарии, SEO из коробки.
- **Ролей сложнее `ADMIN`** — один уровень доступа достаточен для масштаба.
- **Пользователей в БД** — админы задаются переменной окружения; смена пароля = рестарт.
- **Очередей/брокеров** — Telegram-уведомления шлются синхронно с таймаутами и best-effort.
- **Redis** — кэш и rate limit живут в памяти процесса; верно до тех пор, пока инстанс один.
