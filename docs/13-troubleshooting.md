# Troubleshooting

Типовые проблемы и решения. Сгруппировано по симптому.

## Приложение не стартует

### `app.admins is empty. Example: APP_ADMINS=admin:pass,...`
**Причина:** не задана обязательная переменная `APP_ADMINS`.
**Решение:** задай её. Локально — `APP_ADMINS=admin:admin ./mvnw spring-boot:run`. В Docker — пропиши в `.env` (compose иначе тоже не стартует с явной ошибкой).

### `Bad admin entry: ... (need user:pass)`
**Причина:** формат `APP_ADMINS` нарушен. Нужно `user:pass`, несколько — через запятую: `admin:pass1,alex:pass2`.
**Решение:** проверь, что в каждой паре ровно одно двоеточие-разделитель и нет лишних пробелов/символов.

### `Found more than one migration with version N`
**Причина:** в `target/classes/db/migration` остались две миграции с одной версией (например, после переименования файла осталась старая копия в сборке).
**Решение:**
```bash
rm -rf target/classes/db    # или
./mvnw clean
```
В исходниках (`src/main/resources/db/migration`) тоже не должно быть двух файлов `V<N>__`.

### `site_settings row not found (id=1). Check Flyway migration.`
**Причина:** не отработала миграция `V3`, вставляющая дефолтную строку настроек (или строку удалили вручную).
**Решение:** проверь, что Flyway применил все миграции (`select * from flyway_schema_history`). При ручном удалении строки — восстанови: `insert into site_settings (id, hero_title, hero_subtitle) values (1, 'Стихи на заказ', '...') on conflict do nothing;`.

### `A component required a bean of type '...CacheManager' that could not be found`
**Причина:** `@EnableCaching` без объявленного `CacheManager` (в Spring Boot 4 cache-автоконфигурация — в отдельном стартере).
**Решение:** должен присутствовать `config/CacheConfig` с бином `ConcurrentMapCacheManager`. Если удалял — верни.

### Не подключается к БД / `Connection refused` на 5432
**Причина:** Postgres не запущен или недоступен по указанному URL.
**Решение:**
- Docker: `docker compose up -d db`, дождись healthy (`docker compose ps`).
- Локально: проверь, что Postgres слушает 5432, а `SPRING_DATASOURCE_URL/USERNAME/PASSWORD` верны.

### `permission denied: ./mvnw`
**Решение:** `sh mvnw ...` или `chmod +x mvnw`.

### `initdb: invalid locale settings` (при локальной установке Postgres)
**Решение:** задай локаль перед initdb: `export LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8`.

## Поведение приложения

### Новый публичный URL отдаёт редирект на `/admin/login`
**Причина:** маршрут не добавлен в permit-list — его ловит `anyRequest().denyAll()`.
**Решение:** добавь путь в `SecurityConfig` (см. [05-security.md](05-security.md) и [11-development.md](11-development.md)).

### POST формы возвращает 403
**Причина:** отсутствует/невалиден CSRF-токен, либо потеряна сессионная cookie (`JSESSIONID`).
**Решение:** убедись, что форма рендерится через `th:action` (тогда токен вставляется автоматически), а клиент сохраняет cookie между GET (формы) и POST. В curl — используй `-c`/`-b` с одним cookie-jar.

### POST формы возвращает 429
**Причина:** сработал rate limit (по умолчанию >5 отправок с IP за 10 минут).
**Решение:** это ожидаемо. Для разработки подними лимит (`RATE_LIMIT_MAX`) или подожди окно. Если 429 у всех пользователей сразу — proxy не передаёт `X-Forwarded-For`, и все клиенты видятся одним IP (см. [08-antispam.md](08-antispam.md)).

### Добавленный стих не виден на сайте
**Причина:** статус `Черновик`. На витрину попадают только `Опубликовано`.

### Отзыв не появляется на сайте
**Причина:** отзывы создаются в статусе `PENDING`, публикуются только после «Принять» в админке.

### Список стихов в админке пуст, хотя стихи есть
**Историческая причина:** шаблон читал `${items}` вместо `${poems}` — исправлено. Если воспроизводится — проверь, что имя атрибута в контроллере совпадает с шаблоном.

### Dropdown статусов на карточке заказа пуст
**Историческая причина:** шаблон читал `${allStatuses}`, а контроллер кладёт `statuses` — исправлено. При аналогичных проблемах сверь имена model-атрибутов контроллера и шаблона.

### Изменил настройки/контакты — на сайте старое
**Причина:** эти данные кэшируются. Сброс происходит автоматически при сохранении через админку (`@CacheEvict`).
**Решение:** если правил данные **в обход** сервиса (прямо в БД) — кэш не сбросился. Перезапусти приложение или меняй через админку.

### Ссылка в контактах не сохраняется (`?error`)
**Причина:** `href` не прошёл валидацию схемы.
**Решение:** ссылка должна начинаться с `http://`, `https://`, `mailto:` или `tel:`.

### Не приходят Telegram-уведомления
**Проверь по порядку:**
1. Заданы **оба** — `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`? При старте в логе: `Telegram notifications disabled ...` означает, что нет.
2. Нажал ли получатель «Start» в чате с ботом (для личных сообщений Bot API не может писать первым).
3. Верный ли `chat_id` (для групп — с минусом).
4. Смотри `WARN` в логах: `Telegram notification failed: ...` — там причина (таймаут, 401 неверный токен, 400 неверный chat_id).

### Дата отображается странно / не в том поясе
**Контекст:** всё хранится в UTC (`timestamptz`, `hibernate.jdbc.time_zone=UTC`). Шаблоны форматируют без явной зоны. Если нужен локальный пояс в отображении — добавь форматирование с зоной в шаблоне.

## Диагностика

```bash
# логи приложения
docker compose logs -f app

# состояние контейнеров
docker compose ps

# применённые миграции
docker exec -i poems_db psql -U poems poems -c "select version, description, success from flyway_schema_history order by installed_rank;"

# здоровье
curl -s http://127.0.0.1:8081/actuator/health
```

Если симптома здесь нет — смотри лог старта целиком (там Spring печатает причину падения контекста) и сверяйся с соответствующим разделом docs.
