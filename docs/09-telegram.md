# Telegram-уведомления

Уведомления о **новых заявках** и **новых отзывах** приходят в Telegram. Функция опциональна: без токена приложение работает как обычно.

## Компоненты

| Класс | Роль |
|---|---|
| `integration/telegram/TelegramNotifier` | отправка сообщения в Bot API |
| `integration/telegram/TelegramMessageFormatter` | формирование текста сообщения |

Точки вызова:
- `OrderService.createFromForm` → `formatter.newOrder(saved)` → `notifier.send(...)`
- `ReviewService.createPending` → `formatter.newReview(saved)` → `notifier.send(...)`

## Настройка

1. В Telegram напиши [@BotFather](https://t.me/BotFather) → `/newbot` → получи **токен** вида `123456:ABC-DEF...`.
2. Узнай **chat id** получателя. Простой способ: написать боту [@userinfobot](https://t.me/userinfobot) — он покажет твой id. Для группы — добавь бота в группу и используй id группы (с минусом).
3. Задай переменные окружения:

```bash
TELEGRAM_BOT_TOKEN=123456:ABC-DEF...
TELEGRAM_CHAT_ID=987654321
```

В `docker-compose.yml` они уже проброшены (пустые по умолчанию) — заполни их в `.env`. Перезапусти приложение.

4. Чтобы бот мог тебе писать — один раз нажми «Start» в чате с ботом (для личных сообщений).

## Как работает

```java
this.enabled = !botToken.isBlank() && !chatId.isBlank();
```

- **Оба** параметра должны быть заданы, иначе уведомления выключены (в лог при старте: `Telegram notifications disabled ...`).
- Запрос: `POST https://api.telegram.org/bot<token>/sendMessage` с JSON `{chat_id, text}`.
- Таймауты: 3 с на соединение, 5 с на чтение (`SimpleClientHttpRequestFactory`).
- Всё обёрнуто в `try/catch`: при любой ошибке (Telegram недоступен, неверный токен, таймаут) пишется `WARN` в лог, но исключение **не пробрасывается** — заявка/отзыв уже сохранены и сабмит не ломается.

## Формат сообщений

**Новая заявка:**
```
🆕 Новая заявка #42
Имя: Иван
Телефон: +79990001122
Соцсеть: @ivan

Хочу стих про кота на юбилей...
```
(поле «Соцсеть» — только если заполнено; описание обрезается до 500 символов).

**Новый отзыв:**
```
💬 Новый отзыв на модерацию #17
Имя: Мария
Telegram: @maria

Чудесный стих, спасибо!
```
(строка Telegram — только если ник указан; текст обрезается до 500 символов).

Сообщения отправляются обычным текстом (без HTML/Markdown-разметки) — безопасно при любом содержимом.

## Дизайн-решения

- **Синхронно, а не через очередь.** Жёсткие таймауты ограничивают худший случай 8 секундами задержки сабмита. Очередь/ретраи не нужны: уведомление дублируется списком в админке, потеря одного сообщения некритична.
- **Best-effort.** Доступность Telegram не должна влиять на приём заявок — поэтому ошибки только логируются.
- **Отзывы уведомляют на стадии PENDING** — чтобы быстро отмодерировать, не заходя в админку постоянно.

## Возможные улучшения

- Уведомления о смене статуса заказа.
- Кнопки модерации прямо в Telegram (через Bot API inline keyboard + webhook) — но это уже полноценная интеграция, не best-effort.
- Асинхронная отправка (`@Async`), если таймаут начнёт мешать.
